## 2.2 Temel Veri Tabanı Komutları (CRUD ve Temel Sorgular)

2.1’de, veri tabanından veri okumak için kullanılan `SELECT` ifadesinin temel yapısını ele aldık.  
Bu bölümde, ilişkisel veri tabanı ile günlük hayatta en sık kullanılan komutları bütünsel bir çerçevede işleyeceğiz:

- SELECT – Veri okuma  
- INSERT – Veri ekleme  
- UPDATE – Veri güncelleme  
- DELETE – Veri silme  
- ORDER BY – Sıralama  
- LIMIT – Kayıt sayısını sınırlama  
- COUNT / SUM / AVG – Basit hesaplamalar  
- GROUP BY – Gruplama  

Amaç, SQLAlchemy’ye geçmeden önce temel SQL işlemlerine karşı tereddüdü ortadan kaldırmaktır.

---

### 2.2.1 Örnek Tablo: students

Bu bölüm boyunca aşağıdaki tabloyu kullanacağız:

| id | name   | age | department | score |
| -- | ------ | --- | ---------- | ----- |
| 1  | Ali    | 20  | Software   | 80    |
| 2  | Ayşe   | 22  | Electrical | 90    |
| 3  | Mehmet | 21  | Software   | 75    |
| 4  | Zeynep | 23  | Mechanical | 88    |

Tablonun şeması şu şekilde özetlenebilir:

```text
students(id, name, age, department, score)
```

---

### 2.2.2 SELECT – Veri Okuma (Özet)

Tüm tabloyu getirmek:

```sql
SELECT *
FROM students;
```

Belirli sütunları getirmek:

```sql
SELECT name, department
FROM students;
```

Filtreleme:

```sql
SELECT *
FROM students
WHERE department = 'Software';
```

Bu temel kullanım, 2.1’de ayrıntılı olarak ele alınmıştır; burada diğer komutlarla bütünlüğü görmek amacıyla kısaca özetlenmiştir.

---

### 2.2.3 INSERT – Veri Ekleme

Yeni bir kayıt eklemek için `INSERT` komutu kullanılır.

Tek kayıt ekleme:

```sql
INSERT INTO students (name, age, department, score)
VALUES ('Can', 19, 'Software', 85);
```

Birden fazla kayıt ekleme:

```sql
INSERT INTO students (name, age, department, score)
VALUES
    ('Ece', 20, 'Mechanical', 78),
    ('Burak', 22, 'Electrical', 92);
```

Burada dikkat edilmesi gereken noktalar:

- Sütun sırası ile `VALUES` kısmındaki değer sırası eşleşmelidir.  
- Primary key alanı otomatik artan ise, genellikle VALUES kısmında ayrıca belirtilmez.  

---

### 2.2.4 UPDATE – Veri Güncelleme

Var olan bir kaydı güncellemek için `UPDATE` komutu kullanılır.

Belirli bir öğrencinin notunu güncellemek:

```sql
UPDATE students
SET score = 95
WHERE id = 2;
```

Önemli bir uyarı:

```sql
UPDATE students
SET score = 100;
```

Bu sorguda `WHERE` ifadesi bulunmamaktadır. Sonuç olarak:

- Tablodaki tüm kayıtların `score` değeri 100’e güncellenir.  

Bu nedenle, UPDATE komutu kullanılırken `WHERE` koşulu dikkatle tanımlanmalıdır.

---

### 2.2.5 DELETE – Veri Silme

Kayıt silmek için `DELETE` komutu kullanılır.

Belirli bir öğrenciyi silmek:

```sql
DELETE FROM students
WHERE id = 3;
```

Tüm tabloyu boşaltmak:

```sql
DELETE FROM students;
```

Bu tür bir kullanım, tablodaki tüm kayıtların silinmesine neden olur ve geri dönüşü zor bir işlemdir.  
Özellikle üretim ortamında, DELETE komutları dikkatle ve mümkünse transaction içinde kullanılmalıdır.

---

### 2.2.6 ORDER BY – Sıralama

Sonuçları belirli bir sütuna göre sıralamak için `ORDER BY` kullanılır.

Yaşa göre küçükten büyüğe sıralama:

```sql
SELECT *
FROM students
ORDER BY age ASC;
```

Puanı yüksekten düşüğe sıralama:

```sql
SELECT *
FROM students
ORDER BY score DESC;
```

`ASC` (artan) ve `DESC` (azalan) anahtar kelimeleri, sıralama yönünü belirler.

---

### 2.2.7 LIMIT – Kayıt Sayısını Sınırlama

Sonuç setinden yalnızca belirli sayıda kayıt getirmek için `LIMIT` ifadesi kullanılır.

Örnek:

```sql
SELECT *
FROM students
ORDER BY score DESC
LIMIT 2;
```

Bu sorgu:

- En yüksek puana sahip ilk iki öğrenciyi getirir.  

Gerçek uygulamalarda `LIMIT`, sayfalama (pagination) senaryolarında yoğun şekilde kullanılır.

---

### 2.2.8 COUNT, SUM, AVG – Basit Hesaplamalar

SQL, temel istatistiksel işlemler için de kullanılabilir.

Toplam öğrenci sayısı:

```sql
SELECT COUNT(*)
FROM students;
```

Ortalama puan:

```sql
SELECT AVG(score)
FROM students;
```

Toplam puan:

```sql
SELECT SUM(score)
FROM students;
```

Bu fonksiyonlar, özellikle raporlama ve özet bilgi oluşturma süreçlerinde kullanılır.

---

### 2.2.9 GROUP BY – Gruplama

Belirli bir sütuna göre gruplama işlemi için `GROUP BY` kullanılır. Grup bazında hesaplama yapmak istediğimizde tercih edilir.

Örnek: Bölüme göre ortalama puan hesaplamak:

```sql
SELECT
    department,
    AVG(score) AS average_score
FROM students
GROUP BY department;
```

Bu sorgu:

- Her bir bölüm için ayrı satır üretir.  
- Her bölümdeki öğrencilerin ortalama puanını hesaplar.  

`GROUP BY`, `COUNT`, `SUM`, `AVG` gibi fonksiyonlarla birlikte sıkça kullanılır.

---

### 2.2.10 JOIN – Tabloları Birleştirme (Giriş)

İlişkisel veri tabanlarında çoğu zaman birden fazla tabloyu aynı sorgu içerisinde birleştirmek gerekir.  
Bu işlem için `JOIN` yapıları kullanılır.

Örnek olarak, aşağıdaki gibi bir `departments` tablosu tanımlı olsun:

| id | name      |
| -- | --------- |
| 1  | Software  |
| 2  | Electrical|
| 3  | Mechanical|

`students` tablosunun `department` yerine `department_id` tuttuğunu varsayalım. Bu durumda, öğrencileri bölüm adları ile birlikte listelemek için:

```sql
SELECT
    s.name,
    d.name AS department_name
FROM students s
INNER JOIN departments d
    ON s.department_id = d.id;
```

JOIN kavramı ilerleyen başlıklarda (özellikle 2.6 ve ilişki türleri bölümlerinde) daha ayrıntılı ele alınacaktır. Bu bölümde amaç, temel fikre aşinalık kazandırmaktır.

---

### 2.2.11 Komutların Özet Tablosu

| Komut    | Temel Amaç              |
| -------- | ------------------------|
| SELECT   | Veri okumak             |
| INSERT   | Yeni veri eklemek       |
| UPDATE   | Mevcut veriyi güncellemek |
| DELETE   | Veri silmek             |
| ORDER BY | Sonuçları sıralamak     |
| LIMIT    | Kayıt sayısını sınırlamak |
| COUNT    | Kayıt sayısını hesaplamak |
| SUM      | Toplam hesaplamak       |
| AVG      | Ortalama hesaplamak     |
| GROUP BY | Gruplayarak özetlemek   |
| JOIN     | Tabloları birleştirmek  |

---

### 2.2.12 Bölüm Özeti

Bu bölümle birlikte:

- Temel SQL komutlarının (CRUD ve yardımcı komutlar) ne amaçla kullanıldığı,  
- Sorguların nasıl okunacağı ve yazılacağı,  
- Basit raporlama ve özet bilgilerin nasıl elde edileceği  

örnekler üzerinden gösterilmiştir.

Bir sonraki adımlarda, bu komutların:

- İlişkisel veri modelleri ile nasıl bir araya geldiği,  
- Veri tipleri, constraint’ler ve şema tasarımıyla nasıl bütünleştiği,  

ele alınacak ve ardından SQLAlchemy ile bu yapıların Python sınıfları üzerinden nasıl temsil edildiği gösterilecektir.
