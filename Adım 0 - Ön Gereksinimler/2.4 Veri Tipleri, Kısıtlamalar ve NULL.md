## 2.4 Veri Tipleri, Kısıtlamalar ve NULL Mantığı

İlişkisel veri tabanlarında her sütun, belirli bir türde veri tutmak üzere tanımlanır ve bu sütunlar üzerinde çeşitli kurallar (kısıtlamalar) bulunur.  
Bu bölümde üç temel başlık ele alınacaktır:

1. Veri tipleri (Data Types)  
2. Kısıtlamalar (Constraints)  
3. NULL kavramı  

Amaç, tablo tasarlarken doğru veri tipi ve kural seçimini yapabilecek seviyeye gelmektir.

---

### 2.4.1 Veri Tipi Nedir?

Bir sütunun veri tipi, o sütuna hangi türde verilerin yazılabileceğini belirler.  
Doğru veri tipi seçimi:

- Performans,  
- Veri bütünlüğü,  
- Okunabilirlik  

açısından kritik öneme sahiptir.

Yanlış veri tipi seçimleri ise:

- Gereksiz bellek kullanımı,  
- Hatalı veri girişi,  
- Karmaşık dönüşüm ihtiyacı  

gibi sorunlara yol açabilir.

---

### 2.4.2 Temel Veri Tipleri

#### INTEGER

Tam sayı değerlerini saklamak için kullanılır.

```sql
age INTEGER
```

Örnek kullanım alanları:

- Yaş  
- Stok miktarı  
- Adet bilgisi  
- Otomatik artan kimlik (id)  

#### VARCHAR(n)

Belirli bir uzunluğa kadar kısa metin saklamak için kullanılır.

```sql
name VARCHAR(100)
```

Burada `n`, maksimum karakter sayısını temsil eder.

Örnek kullanım alanları:

- İsim  
- E-posta adresi  
- Şehir adı  

#### TEXT

Uzun metin saklamak için kullanılır.

```sql
description TEXT
```

Örnek kullanım alanları:

- Açıklama alanları  
- Blog içerikleri  
- Kullanıcı biyografileri  

#### BOOLEAN

Mantıksal (doğru/yanlış) değerleri saklamak için kullanılır.

```sql
is_active BOOLEAN
```

Örnek kullanım alanları:

- Kullanıcı aktif mi?  
- Kayıt onaylı mı?  

#### DATE / TIMESTAMP

Tarih ve zaman bilgilerini saklamak için kullanılır.

```sql
created_at TIMESTAMP
```

Örnek kullanım alanları:

- Kayıt oluşturulma tarihi  
- Sipariş tarihi  
- Son giriş zamanı  

---

### 2.4.3 Doğru Veri Tipi Seçiminin Önemi

Yanlış veri tipi seçimine basit bir örnek:

- Telefon numarası için `INTEGER` tipi seçildiğinde:  
  - Başındaki `0` kaybedilebilir.  
  - Telefon numarası üzerinde matematiksel işlem yapılabilir hâle gelir, bu istenmez.  

Bu nedenle telefon numarası gibi alanlar genellikle:

```sql
phone VARCHAR(20)
```

şeklinde tanımlanır.

Benzer şekilde, para birimi için tam sayı veya uygun hassasiyetli sayısal tiplerin tercih edilmesi gerekir.

---

### 2.4.4 Kısıtlamalar (Constraints) Nedir?

Kısıtlamalar, veri tabanının tablo sütunları üzerine koyduğu kurallardır.  
Amaç, hatalı veya uygunsuz verinin tabloya yazılmasını engelleyerek veri bütünlüğünü korumaktır.

Basit bir ifade ile:

> “Bu tabloya şu kurallara uymayan veri yazılmamalıdır.”

---

### 2.4.5 Temel Kısıtlama Türleri

#### NOT NULL

Sütunun boş bırakılmasına izin verilmez.

```sql
name VARCHAR(100) NOT NULL
```

Bu tanım, her kayıtta `name` alanının dolu olmasını zorunlu kılar.

#### UNIQUE

Sütundaki değerlerin tekrar etmesine izin verilmez.

```sql
email VARCHAR(255) UNIQUE
```

Bu tanım, aynı e-posta adresiyle ikinci bir kullanıcının kayıt olmasını engeller.

#### PRIMARY KEY

- Benzersizdir.  
- NULL olamaz.  
- Genellikle tablo için otomatik bir index oluşturur.  

Örnek:

```sql
id INTEGER PRIMARY KEY
```

#### FOREIGN KEY

Başka bir tablodaki primary key alanına referans verir; tablolar arası ilişkiyi temsil eder.

```sql
user_id INTEGER REFERENCES users(id)
```

Bu tanım, `user_id` değerinin, `users` tablosundaki geçerli bir `id` ile eşleşmesini zorunlu kılar.

#### DEFAULT

Sütuna veri girilmediğinde kullanılacak varsayılan değeri tanımlar.

```sql
is_active BOOLEAN DEFAULT TRUE
```

Bu tanım, aksi belirtilmediği sürece `is_active` alanını `TRUE` olarak ayarlar.

#### CHECK

Belirli bir mantıksal koşulu zorunlu kılar.

```sql
age INTEGER CHECK (age >= 0)
```

Bu tanım, negatif yaş değerlerinin tabloya yazılmasını engeller.

---

### 2.4.6 Neden Kısıtlama Kullanılır?

Kısıtlamalar olmazsa, aşağıdaki türden hatalı veriler sistemde birikebilir:

- Negatif stok değerleri  
- Boş isim alanları  
- Aynı kullanıcıya ait birden fazla e-posta kaydı  

Uygulama katmanında kontroller bulunsa bile, veri tabanı düzeyinde kısıtlama tanımlamak:

- Hatalara karşı ek bir güvenlik katmanı sağlar.  
- Farklı uygulamaların aynı veri tabanını kullanması durumunda tutarlılığı korur.  

---

### 2.4.7 NULL Kavramı

NULL, veri tabanında “bilinmiyor” veya “tanımsız” anlamına gelir.  
Önemli bir nokta:

> NULL, boş string (`""`) veya sıfır (`0`) ile aynı şey değildir.

Örnek:

- Kullanıcı telefon numarası vermemiş olabilir → `telefon` alanı `NULL`.  
- Kullanıcının telefon alanı vardır, ancak boş string (`""`) girilmiştir → farklı bir durumdur.  

Bu iki durumun sistem ihtiyacına göre farklı anlamları olabilir.

---

### 2.4.8 NULL ile Sorgulama

NULL değerler, normal karşılaştırma operatörleriyle (`=`, `!=` vb.) kontrol edilmez.

Yanlış kullanım:

```sql
WHERE phone = NULL
```

Doğru kullanım:

```sql
WHERE phone IS NULL
```

Benzer şekilde, NULL olmayan değerleri filtrelemek için:

```sql
WHERE phone IS NOT NULL
```

kullanılır.

NULL, matematiksel işlemler ve karşılaştırmalar açısından özel kurallara tabidir; bu nedenle sorgularda dikkatle ele alınmalıdır.

---

### 2.4.9 SQLAlchemy ile Bağlantı

Veri tipi ve kısıtlama kavramları, SQLAlchemy tarafında sütun tanımlarında karşılık bulur:

```python
from sqlalchemy import Column, Integer, String, Boolean

id = Column(Integer, primary_key=True)
name = Column(String(100), nullable=False)
email = Column(String(255), unique=True, nullable=False)
is_active = Column(Boolean, default=True)
phone = Column(String(20), nullable=True)
```

Burada:

- `Integer`, `String`, `Boolean` → veri tiplerini,  
- `primary_key`, `unique`, `nullable`, `default` → veri tabanı kısıtlamalarını,  

temsil eder.  
Bu sayede, SQL düzeyinde tanımladığınız modelin aynısı, ORM seviyesinde sınıf ve alanlar üzerinden ifade edilmiş olur.

Bu bölümde:

- Temel veri tipleri ve hangi alanlar için uygun oldukları,  
- Kısıtlamaların veri bütünlüğü açısından rolü,

ele alınmıştır.

Bu başlık, tablo tasarımı ve ilişkisel modellerin teknik temelini oluşturur. Sonraki bölümler, bu temeli kullanarak:
