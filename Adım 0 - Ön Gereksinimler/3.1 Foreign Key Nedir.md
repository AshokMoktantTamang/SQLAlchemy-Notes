## 3.1 Foreign Key Nedir?

Önceki bölümlerde:

- Primary key kavramını,  
- Temel ilişki türlerini ve  
- Örnek veri modellerini  

gördünüz. Bu bölümde, tablolar arasındaki bağlantıyı teknik olarak kuran yapı olan **foreign key (yabancı anahtar)** kavramını ele alacağız.

---

### 3.1.1 Temel Tanım

**Foreign key**, bir tablodaki bir sütunun, başka bir tablodaki primary key sütununa referans vermesidir.

Başka bir ifadeyle:

> Foreign key, bir kaydın başka bir tablodaki belirli bir kayıtla ilişkili olduğunu ifade eder.

Bu sayede:

- Tablolar arası bağ veri tabanı düzeyinde tanımlanır.  
- Var olmayan kayıtlara referans verilmesi engellenebilir.  
- İlişkisel sorguların temeli atılmış olur.  

---

### 3.1.2 Örnek Senaryo: Kullanıcılar ve Siparişler

Basit bir e-ticaret örneği üzerinden ilerleyelim.

**Users tablosu**

```text
users(id, name)
```

Örnek veriler:

| id | name |
| -- | ---- |
| 1  | Ali  |
| 2  | Ayşe |

**Orders tablosu**

```text
orders(id, user_id, amount)
```

Örnek veriler:

| id | user_id | amount |
| -- | ------- | ------ |
| 1  | 1       | 500    |
| 2  | 1       | 200    |
| 3  | 2       | 800    |

Bu modelde:

- `users.id` → Users tablosunun primary key sütunu,  
- `orders.user_id` → Orders tablosunun foreign key sütunu olarak düşünülebilir.  

`orders.user_id = 1` olan tüm satırlar, `id = 1` olan kullanıcıya (Ali) aittir.

---

### 3.1.3 SQL ile Foreign Key Tanımı

Foreign key, tablo tanımı içerisinde aşağıdaki şekilde belirtilebilir:

```sql
CREATE TABLE users (
    id   INTEGER PRIMARY KEY,
    name VARCHAR(100) NOT NULL
);

CREATE TABLE orders (
    id      INTEGER PRIMARY KEY,
    user_id INTEGER NOT NULL,
    amount  NUMERIC(10, 2) NOT NULL,

    CONSTRAINT fk_orders_users
        FOREIGN KEY (user_id) REFERENCES users(id)
);
```

Bu tanım şunları ifade eder:

- `orders.user_id` sütunu, `users.id` sütununa referans verir.  
- Veri tabanı, `orders.user_id` alanına yazılan değerin `users.id` içinde var olup olmadığını kontrol edebilir.  

Sonuç olarak, sisteme kayıtlı olmayan bir kullanıcı için sipariş oluşturulması engellenmiş olur (kullanılan veri tabanı sistemine ve constraint ayarlarına bağlı olarak).

---

### 3.1.4 Foreign Key ve Veri Bütünlüğü

Foreign key kullanmanın temel amaçları şunlardır:

- **Veri tekrarını azaltmak:** Kullanıcı adı veya diğer bilgileri her sipariş satırında tekrar etmek yerine, bu bilgiler `users` tablosunda tek bir yerde tutulur.  
- **Tutarlılığı korumak:** Kullanıcı silindiğinde, ona bağlı siparişlerin durumu kontrol altına alınabilir (örneğin kaskad silme veya engelleme kuralları).  
- **Hatalı referansları önlemek:** Var olmayan kullanıcıya veya var olmayan kaynağa referans veren kayıtların eklenmesi engellenebilir.  

Bu özellikler, veri tabanı tasarımında foreign key’i merkezi bir kavram hâline getirir.

---

### 3.1.5 Foreign Key ve İlişki Türleri

Foreign key, ilişki türlerinin teknik düzeyde ifade edilmesini sağlar:

- One-to-Many: “Çok” tarafında foreign key bulunur (ör. `orders.user_id`).  
- Many-to-One: Aynı ilişkinin diğer bakış açısıdır (her sipariş bir kullanıcıya aittir).  
- Many-to-Many: Ara tabloda iki ayrı foreign key kullanılarak ifade edilir (ör. `student_courses.student_id` ve `student_courses.course_id`).  

İlişki türleri ayrıntılı olarak 3.2 başlığında ele alınacaktır.

---

### 3.1.6 SQLAlchemy ile Bağlantı

ORM tarafında foreign key tanımları genellikle şu şekilde yapılır:

```python
from sqlalchemy import Column, Integer, ForeignKey
from sqlalchemy.orm import relationship

class User(Base):
    __tablename__ = "users"

    id = Column(Integer, primary_key=True)
    name = Column(String(100), nullable=False)

    orders = relationship("Order")

class Order(Base):
    __tablename__ = "orders"

    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False)
```

Burada:

- `ForeignKey("users.id")` ifadesi, SQL düzeyinde tanımladığımız foreign key kuralının ORM tarafındaki karşılığıdır.  
- `orders` ilişki alanı, 3.2 ve devamındaki başlıklarda ele alınacak ilişki türleri ile birlikte anlam kazanacaktır.

---

### 3.1.7 Bölüm Özeti

Bu bölümde:

- Foreign key’in tanımı ve primary key ile ilişkisi,  
- E-ticaret örneği üzerinden tablo bağlantıları,  
- SQL’de foreign key tanımlama biçimi,  
- Veri bütünlüğü ve ilişki modelindeki rolü,  
- SQLAlchemy tarafındaki temel karşılığı  

ele alınmıştır.  
Bir sonraki başlıkta, foreign key kullanılarak kurulabilen **ilişki türleri** (One-to-One, One-to-Many, Many-to-Many) sistematik şekilde incelenecektir.
